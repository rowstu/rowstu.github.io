Core DevOps Principles

DevOps is a set of practices that combines software development (Dev) and IT operations (Ops) to shorten the development lifecycle while delivering features, fixes, and updates frequently and reliably.

The Three Ways

1. Flow (Systems Thinking)

Focus on the performance of the entire system, not just individual silos.

  • Optimise for global goals, not local departmental goals
  • Reduce batch sizes and work in progress
  • Eliminate constraints and bottlenecks
  • Never pass defects downstream
  • Create smooth, fast flow from development to operations to customer

2. Feedback (Amplify Feedback Loops)

Create fast, constant feedback at all stages.

  • Shorten and amplify feedback loops
  • Create quality at source
  • Embed knowledge where it’s needed
  • Stop the line when defects are found
  • Enable fast detection and recovery

3. Continuous Learning (Culture of Experimentation)

Foster a culture of continual experimentation and learning.

  • Allocate time for improvement
  • Create rituals that reward risk-taking
  • Introduce faults to increase resilience
  • Convert local discoveries into global improvements
  • Learn from both success and failure

Key DevOps Practices

Continuous Integration (CI)

  • Developers integrate code frequently (at least daily)
  • Each integration triggers automated build and tests
  • Fast feedback on code quality
  • Detect integration issues early

Continuous Delivery (CD)

  • Code is always in a deployable state
  • Automated deployment pipeline
  • One-click deployment to any environment
  • Deployment is routine, not an event

Continuous Deployment

  • Every change that passes tests is deployed automatically
  • No manual approval gates to production
  • Requires excellent test coverage and monitoring
  • Fastest path from code to production

Infrastructure as Code (IaC)

  • Manage infrastructure through code, not manual processes
  • Version control for infrastructure
  • Reproducible, consistent environments
  • Enables automation and self-service

Monitoring and Observability

  • Collect metrics, logs, and traces
  • Understand system behaviour
  • Detect issues before users do
  • Enable data-driven decisions

Microservices Architecture

  • Small, independently deployable services
  • Each service owns its data
  • Enables independent scaling and deployment
  • Technology diversity per service

Anti-Patterns to Avoid

Silos

  • “Throw it over the wall” mentality
  • Separate dev and ops teams with no collaboration
  • Blame culture when things go wrong

Manual Processes

  • Manual deployments
  • Manual environment provisioning
  • Manual testing
  • “Snowflake” servers

Infrequent Releases

  • Large batch releases
  • Long-lived feature branches
  • Deployment as a special event
  • Change advisory boards for every change

Lack of Feedback

  • No monitoring
  • No metrics
  • No post-incident reviews
  • Ignoring technical debt

DevOps Maturity Model

Level 1: Initial

  • Manual processes
  • Siloed teams
  • Infrequent releases
  • Reactive operations

Level 2: Managed

  • Some automation
  • Basic CI/CD
  • Defined processes
  • Basic monitoring

Level 3: Defined

  • Consistent automation
  • Full CI/CD pipeline
  • Infrastructure as Code
  • Proactive monitoring

Level 4: Measured

  • Metrics-driven decisions
  • Continuous improvement
  • Self-service platforms
  • Automated security

Level 5: Optimised

  • Continuous deployment
  • Full observability
  • Chaos engineering
  • Innovation culture

Cultural Principles

Blameless Post-Mortems

  • Focus on systems, not individuals
  • Ask “how” not “who”
  • Share findings widely
  • Implement improvements

Shared Responsibility

  • Everyone owns quality
  • Everyone owns uptime
  • Everyone owns security
  • “You build it, you run it”

Psychological Safety

  • Safe to take risks
  • Safe to ask questions
  • Safe to admit mistakes
  • Safe to challenge the status quo

Continuous Improvement

  • Kaizen mindset
  • Regular retrospectives
  • Experimentation time
  • Learn from failures
