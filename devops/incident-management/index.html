Severity Criteria

SEV1 - Critical

  • Complete service unavailable
  • Data loss or corruption
  • Security breach
  • Revenue impact > Â£X/hour

SEV2 - High

  • Major feature unavailable
  • Significant performance degradation
  • Affecting large % of users
  • Workaround difficult or impossible

SEV3 - Medium

  • Minor feature unavailable
  • Performance degradation
  • Affecting small % of users
  • Workaround available

SEV4 - Low

  • Cosmetic issues
  • Minor inconvenience
  • Single user affected
  • Easy workaround

Incident Roles

Incident Commander (IC)

The single point of coordination.

Responsibilities:

  • Declare and close incidents
  • Coordinate response efforts
  • Make decisions
  • Communicate status
  • Escalate when needed
  • Ensure documentation

Not responsible for:

  • Fixing the problem directly
  • Deep technical investigation

Technical Lead

Leads the technical investigation.

Responsibilities:

  • Investigate root cause
  • Coordinate technical response
  • Implement fixes
  • Advise IC on technical decisions

Communications Lead

Manages all communications.

Responsibilities:

  • Update status page
  • Communicate with stakeholders
  • Draft customer communications
  • Manage internal updates

Scribe

Documents everything.

Responsibilities:

  • Record timeline of events
  • Document decisions made
  • Capture actions taken
  • Note who did what
