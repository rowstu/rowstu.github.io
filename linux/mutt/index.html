Mutt – Terminal Email Client

Home / Browse/ Linuxmutt

Mutt is a powerful text-based email client for Linux. It handles IMAP, POP3, and SMTP natively, and with the right configuration can connect to Gmail, Outlook/Exchange (via OAuth2 or app passwords), and any standard mail server. NeoMutt is a maintained fork with extra features — everything here works with both.

Installation

# Debian/Ubuntu
sudo apt install mutt
# or NeoMutt (recommended)
sudo apt install neomutt

# Arch
sudo pacman -S neomutt

# Fedora
sudo dnf install neomutt

For Exchange/Office 365, you’ll also want:

# DavMail - bridges Exchange to IMAP/SMTP
sudo apt install davmail

# Or use hydroxide for ProtonMail
# Or mbsync + mu for offline IMAP

Basic IMAP Configuration

Create ~/.muttrc (or ~/.config/mutt/muttrc):

# ── Identity ──────────────────────────────────────────
set realname = "Your Name"
set from = "you@example.com"

# ── IMAP ──────────────────────────────────────────────
set imap_user = "you@example.com"
set imap_pass = "your-password"
set folder = "imaps://mail.example.com:993/"
set spoolfile = "+INBOX"
set record = "+Sent"
set postponed = "+Drafts"
set trash = "+Trash"

# ── SMTP ──────────────────────────────────────────────
set smtp_url = "smtps://you@example.com@mail.example.com:465/"
set smtp_pass = "your-password"

# ── TLS ───────────────────────────────────────────────
set ssl_force_tls = yes
set ssl_starttls = yes

That’s the minimum to get connected. The rest of this guide covers making it actually pleasant to use.

Gmail

Gmail requires an App Password (if 2FA is enabled) or OAuth2.

With App Password (simplest)

  1. Go to myaccount.google.com → Security → 2-Step Verification → App passwords
  2. Generate a password for “Mail”
set imap_user = "you@gmail.com"
set imap_pass = "xxxx xxxx xxxx xxxx"
set folder = "imaps://imap.gmail.com:993/"
set spoolfile = "+INBOX"
set record = "+[Gmail]/Sent Mail"
set postponed = "+[Gmail]/Drafts"
set trash = "+[Gmail]/Bin"

set smtp_url = "smtps://you@gmail.com@smtp.gmail.com:465/"
set smtp_pass = "xxxx xxxx xxxx xxxx"

set from = "you@gmail.com"
set realname = "Your Name"

Gmail-Specific Tweaks

# Gmail uses labels, not folders — don't move deleted messages
set delete = yes
unset confirmappend
set move = no

# Gmail handles sent mail automatically (server-side copy)
unset record

Microsoft Exchange / Office 365

Exchange is the most complex to set up. There are three approaches, from simplest to most robust.

Option 1: Direct IMAP (Office 365 with App Password)

If your org allows IMAP access and app passwords:

set imap_user = "you@company.com"
set imap_pass = "your-app-password"
set folder = "imaps://outlook.office365.com:993/"
set spoolfile = "+INBOX"
set record = "+Sent Items"
set postponed = "+Drafts"
set trash = "+Deleted Items"

set smtp_url = "smtps://you@company.com@smtp.office365.com:587/"
set smtp_pass = "your-app-password"
set smtp_authenticators = "login"

Note: Microsoft is deprecating basic auth. This may not work for all tenants.

Option 2: OAuth2 (Modern Auth)

NeoMutt supports OAuth2 natively. This is the correct approach for Office 365 tenants that require modern authentication.

First, install the helper script:

# Get the mutt_oauth2.py script (included with NeoMutt or from GitHub)
# Place it somewhere in your PATH
chmod +x /usr/local/bin/mutt_oauth2.py

Create a token file:

# First-time setup — opens a browser for authentication
mutt_oauth2.py ~/.config/mutt/oauth2-token \
  --provider microsoft \
  --client-id "your-azure-app-id" \
  --client-secret "your-azure-secret" \
  --authorize

For the Azure App registration:

  1. Go to portal.azure.com → Azure Active Directory → App registrations
  2. New registration, set redirect URI to http://localhost
  3. Under API permissions, add: IMAP.AccessAsUser.All, SMTP.Send, offline_access
  4. Under Certificates & secrets, create a client secret
  5. Note the Application (client) ID and the secret value
set imap_user = "you@company.com"
set folder = "imaps://outlook.office365.com:993/"
set spoolfile = "+INBOX"
set record = "+Sent Items"
set postponed = "+Drafts"
set trash = "+Deleted Items"

set imap_authenticators = "oauthbearer:xoauth2"
set imap_oauth_refresh_command = "mutt_oauth2.py ~/.config/mutt/oauth2-token --quiet"

set smtp_url = "smtp://you@company.com@smtp.office365.com:587/"
set smtp_authenticators = "oauthbearer:xoauth2"
set smtp_oauth_refresh_command = "mutt_oauth2.py ~/.config/mutt/oauth2-token --quiet"

set from = "you@company.com"
set realname = "Your Name"

Option 3: DavMail Bridge

DavMail runs locally and translates Exchange protocols (EWS/OWA) into standard IMAP/SMTP. Works with on-premises Exchange servers that don’t expose IMAP at all.

# Install and start DavMail
sudo apt install davmail
davmail &

Configure DavMail (GUI or ~/.davmail.properties):

davmail.mode=O365Interactive
davmail.url=https://outlook.office365.com/EWS/Exchange.asmx
davmail.imapPort=1143
davmail.smtpPort=1025
davmail.enableEws=auto

Then point Mutt at localhost:

set imap_user = "you@company.com"
set imap_pass = "your-password"
set folder = "imap://localhost:1143/"
set spoolfile = "+INBOX"
set record = "+Sent Items"
set postponed = "+Drafts"

set smtp_url = "smtp://you@company.com@localhost:1025/"
set smtp_pass = "your-password"

set from = "you@company.com"
set realname = "Your Name"

DavMail handles the Exchange authentication dance. Mutt only sees plain IMAP/SMTP.

ProtonMail (via Proton Bridge)

ProtonMail requires their Bridge app to expose IMAP/SMTP locally:

# Install Proton Bridge (from proton.me)
protonmail-bridge &

set imap_user = "you@proton.me"
set imap_pass = "bridge-generated-password"
set folder = "imap://127.0.0.1:1143/"
set spoolfile = "+INBOX"
set record = "+Sent"
set postponed = "+Drafts"

set smtp_url = "smtp://you@proton.me@127.0.0.1:1025/"
set smtp_pass = "bridge-generated-password"

# Bridge uses self-signed cert
set ssl_verify_host = no
set ssl_verify_dates = no

The default Mutt appearance is spartan. These settings transform the experience.

Index Format (Message List)

# ── Index format ──────────────────────────────────────
# Show: flags, date, from, subject, size
set index_format = " %Z  %<[y?%<[d?%[%H:%M]&%[%b %d]>&%[%b %Y]>  %-20.20L  %s"
set sort = threads
set sort_aux = reverse-last-date-received
set collapse_all = yes

# Date: show time for today, day+month for this year, month+year for older

Pstrstrager (Reading Messages)

# ── Pager ─────────────────────────────────────────────
set pager_index_lines = 10    # Show 10 index lines above the pager
set pager_context = 3         # Lines of context when scrolling
set pager_stop = yes          # Don't skip to next message at end
set tstrstrilde = yes              # Pad empty lines with ~
set markers = no              # No + on wrapped lines

# Wrap long lines
set wrap = 120

# Headers to show (hide the noise)
ignore *
unignore from: to: cc: date: subject: reply-to: x-mailer: user-agent:
unhdr_order *
hdr_order from: to: cc: date: subject:

HTML Email

Most email today contains HTML. Mutt can render it:

# ── HTML handling ─────────────────────────────────────
set mailcap_path = "~/.config/mutt/mailcap"
auto_view text/html
alternative_order text/plain text/enriched text/html

Create ~/.config/mutt/mailcap:

text/html; lynx -dump -assume_charset=%{charset} %s; nametemplate=%s.html; copiousoutput
text/html; w3m -dump -T text/html %s; copiousoutput
text/html; firefox %s &; test=test -n "$DISPLAY"

Install a text-mode browser: sudo apt install lynx or w3m. Mutt uses the first matching entry — lynx renders HTML inline, firefox opens it in a browser.

For better HTML rendering, try pandoc:

text/html; pandoc -f html -t plain --wrap=auto %s; copiousoutput

Address Book / Contacts

Using abook

sudo apt install abook

# Query abook for addresses when composing (Tab to autocomplete)
set query_command = "abook --mutt-query '%s'"

# Add sender to abook with a macro
macro index,pager a "<pipe-message>abook --add-email-quiet<return>" "Add sender to abook"

Using khard (CardDAV)

For syncing contacts from a CardDAV server (Nextcloud, iCloud, etc.):

sudo apt install khard vdirsyncer

set query_command = "khard email --parsable '%s'"

Offline with mbsync + notmuch

For working offline or with very large mailboxes, sync mail locally:

sudo apt install isync notmuch

~/.mbsyncrc:

IMAPAccount work
Host outlook.office365.com
Port 993
User you@company.com
PassCmd "pass email/work"
SSLType IMAPS

IMAPStore work-remote
Account work

MaildirStore work-local
Path ~/mail/work/
Inbox ~/mail/work/INBOX
SubFolders Verbatim

Channel work
Far :work-remote:
Near :work-local:
Patterns *
Create Both
SyncState *
Expunge Both

# Sync all mail
mbsync -a

# Index with notmuch
notmuch new

Then point Mutt at the local maildir:

set folder = "~/mail/work"
set mbox_type = Maildir
set spoolfile = "+INBOX"
set record = "+Sent Items"
set postponed = "+Drafts"

# Use notmuch for search (much faster than IMAP search)
set nm_default_url = "notstrstrsmstrstrtp:///home/you/mail/work"
macro index \\ "<vfolder-from-query>" "Search with notmuch"

Password Security

Don’t put passwords in plain text in your muttrc.

Using pass (password-store)

# Store the password
pass insert email/work

# In muttrc, use a command to retrieve it
set imap_pass = `pass email/work`
set smtp_pass = `pass email/work`

Using GPG-encrypted file

echo "your-password" | gpg --encrypt -r you@email.com -o ~/.config/mutt/pass.gpg

set imap_pass = `gpg --quiet --batch --decrypt ~/.config/mutt/pass.gpg`

Using secret-tool (GNOME Keyring)

secret-tool store --label="Email Work" service mutt account work

set imap_pass = `secret-tool lookup service mutt account work`

Multiple Accounts

Use folder hooks to switch settings per-account:

# ── Account: Personal ────────────────────────────────
source ~/.config/mutt/personal.muttrc

# ── Account switching ────────────────────────────────
folder-hook "imaps://imap.gmail.com" "source ~/.config/mutt/personal.muttrc"
folder-hook "imaps://outlook.office365.com" "source ~/.config/mutt/work.muttrc"

# Quick switch macros
macro index <f2> "<sync-mailbox><enter-command>source ~/.config/mutt/personal.muttrc<enter><change-folder>!<enter>" "Switch to personal"
macro index <f3> "<sync-mailbox><enter-command>source ~/.config/mutt/work.muttrc<enter><change-folder>!<enter>" "Switch to work"

Each account file (personal.muttrc, work.muttrc) contains the full set from, set folder, set smtp_url, etc. for that account.

Recommended File Structure

~/.config/mutt/
  muttrc              Main config (or ~/.muttrc)
  personal.muttrc     Personal account settings
  work.muttrc         Work account settings
  signature           Email signature
  mailcap             MIME type handlers
  aliases             Address aliases
  colours.muttrc      Colour scheme (source from muttrc)

Keep each concern in its own file and source them from your main muttrc:

source ~/.config/mutt/colours.muttrc
source ~/.config/mutt/personal.muttrc
