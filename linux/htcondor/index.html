HTCondor: High Throughput Computing

HTCondor is a workload management system for compute-intensive jobs. It manages job queuing, scheduling, resource management, and monitoring across distributed computing resources.

Installation

Debian/Ubuntu

# Add HTCondor repository
curl -fsSL https://research.cs.wisc.edu/htcondor/repo/keys/HTCondor-current-Key | sudo gpg --dearmor -o /usr/share/keyrings/htcondor-archive-keyring.gpg

echo "deb [signed-by=/usr/share/keyrings/htcondor-archive-keyring.gpg] https://research.cs.wisc.edu/htcondor/repo/ubuntu/current $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/htcondor.list

sudo apt update
sudo apt install htcondor

RHEL/CentOS/Rocky

# Add HTCondor repository
sudo yum install https://research.cs.wisc.edu/htcondor/repo/current/htcondor-release-current.el$(rpm -E %rhel).noarch.rpm

sudo yum install condor

Start Services

sudo systemctl enable condor
sudo systemctl start condor

Submit File Basics

A submit file (.sub) describes jobs to HTCondor.

Minimal Example

# job.sub
executable = my_script.sh
output     = job.out
error      = job.err
log        = job.log
queue

Common Submit File Directives

# Executable and arguments
executable = /path/to/program
arguments  = arg1 arg2 arg3

# Input/Output files
input      = input.txt
output     = output.$(Cluster).$(Process).out
error      = error.$(Cluster).$(Process).err
log        = condor.log

# File transfer
transfer_input_files  = data.csv, config.ini
transfer_output_files = results.csv
should_transfer_files = YES
when_to_transfer_output = ON_EXIT

# Resource requirements
request_cpus   = 4
request_memory = 8GB
request_disk   = 10GB

# Environment
environment = "PATH=/usr/bin VAR=value"
getenv      = True

# Notifications
notification = Complete
notify_user  = user@example.com

# Job requirements
requirements = (OpSys == "LINUX") && (Arch == "X86_64")
rank         = Memory

# Queue directive
queue

Submitting Jobs

Submit a Job

condor_submit job.sub

Submit Multiple Jobs

# Run 100 instances
executable = process.sh
arguments  = $(Process)
output     = out.$(Process).txt
queue 100

# Different arguments per job
executable = analyse.sh
arguments  = $(item)
queue item from arguments.txt

# Iterate over files
executable = process.sh
arguments  = $(file)
transfer_input_files = $(file)
queue file matching data/*.csv

# Multiple variables
executable = run.sh
arguments  = $(a) $(b)
queue a, b from (
  1, alpha
  2, beta
  3, gamma
)

Job Management Commands

Check Job Status

# View your jobs
condor_q

# View all jobs in the pool
condor_q -all

# Detailed job info
condor_q -long <job_id>

# Specific attributes
condor_q -af ClusterId ProcId JobStatus RemoteHost

# Analyse why jobs aren't running
condor_q -better-analyze <job_id>

# Watch jobs (refreshes)
condor_q -watch

Job Status Codes

CodeStatus

2Running
3Removed
4Completed
5Held

Manage Jobs

# Remove a job
condor_rm <job_id>

# Remove all your jobs
condor_rm -all

# Hold a job
condor_hold <job_id>

# Release a held job
condor_release <job_id>

# Change job priority
condor_prio -p 10 <job_id>

# Edit a job attribute
condor_qedit <job_id> RequestMemory 16000

# Suspend a job
condor_suspend <job_id>

# Continue a suspended job
condor_continue <job_id>

# Vacate a job (evict, will restart)
condor_vacate_job <job_id>

Universes

Vanilla Universe (Default)

Standard jobs with file transfer.

universe = vanilla
executable = my_program
queue

Docker Universe

Run jobs in Docker containers.

universe = docker
docker_image = python:3.11
executable = /usr/bin/python3
arguments = script.py
transfer_input_files = script.py
queue

Container Universe

Run jobs in Apptainer/Singularity containers.

universe = container
container_image = /path/to/container.sif
executable = /app/run.sh
queue

Java Universe

Run Java applications.

universe = java
executable = MyClass.class
jar_files = myapp.jar,lib.jar
queue

Scheduler Universe

Run on the submit machine (for DAG workflows).

universe = scheduler
executable = local_script.sh
queue

Local Universe

Run on the submit machine as the user.

universe = local
executable = local_task.sh
queue

DAGMan: Workflow Management

DAGMan manages job dependencies as Directed Acyclic Graphs.

DAG File Example

# workflow.dag

# Define jobs
JOB A job_a.sub
JOB B job_b.sub
JOB C job_c.sub
JOB D job_d.sub

# Define dependencies (PARENT -> CHILD)
PARENT A CHILD B C
PARENT B C CHILD D

# Optional: Scripts
SCRIPT PRE A pre_a.sh
SCRIPT POST D post_d.sh

# Optional: Retry failed jobs
RETRY B 3

# Optional: Variables
VARS A input="data1.csv"
VARS B input="data2.csv"

Submit a DAG

condor_submit_dag workflow.dag

DAG Commands

# Check DAG status
condor_q -dag

# Remove a DAG (and all jobs)
condor_rm <dag_job_id>

# Rescue a failed DAG (creates .rescue file)
condor_submit_dag -force workflow.dag

Configuration

View Configuration

# All config
condor_config_val -dump

# Specific parameter
condor_config_val COLLECTOR_HOST

# Configuration sources
condor_config_val -config

Common Configuration Parameters

# /etc/condor/condor_config.local

# Pool settings
CONDOR_HOST = central-manager.example.com
COLLECTOR_HOST = $(CONDOR_HOST):9618

# Allow specific users to submit
ALLOW_WRITE = *.example.com

# Resource limits
NUM_CPUS = 8
MEMORY = 32000
DISK = 100000000

# Slot configuration
NUM_SLOTS = 1
NUM_SLOTS_TYPE_1 = 1
SLOT_TYPE_1 = cpus=100%, memory=100%, disk=100%
SLOT_TYPE_1_PARTITIONABLE = TRUE

# Job limits
MAX_JOBS_RUNNING = 1000

# Security
SEC_DEFAULT_AUTHENTICATION = REQUIRED
SEC_DEFAULT_AUTHENTICATION_METHODS = FS, PASSWORD, SSL

Reconfigure HTCondor

# Reconfigure all daemons
sudo condor_reconfig

# Reconfigure specific machine
sudo condor_reconfig <machine>

Useful Submit File Patterns

Job with Dependencies

executable = step2.sh
transfer_input_files = step1_output.dat
queue

Periodic Jobs

executable = monitor.sh
on_exit_remove = false
periodic_release = (CurrentTime - EnteredCurrentStatus) > 300
queue

GPU Jobs

executable = gpu_task.sh
request_gpus = 1
requirements = (CUDAGlobalMemoryMb >= 8000)
+CUDACapability = 7.0
queue

Memory-Intensive Jobs

executable = big_memory.sh
request_memory = 64GB
requirements = (Memory >= 65536)
queue

Interactive Job

condor_submit -interactive

Or in submit file:

executable = /bin/bash
arguments = -i
queue

Job Wrapper Script

executable = wrapper.sh
arguments = real_program.py $(args)
transfer_input_files = real_program.py
queue args from params.txt

Troubleshooting

Common Issues

Job stuck in Idle:

condor_q -better-analyze <job_id>

Check why job was held:

condor_q -hold -af HoldReason

View job log:

cat job.log

Check daemon logs:

tail -f /var/log/condor/SchedLog
tail -f /var/log/condor/StartLog
tail -f /var/log/condor/CollectorLog

Test job requirements:

condor_status -constraint 'Memory > 8000 && Cpus >= 4'

Useful Debugging

# Check pool resources
condor_status -total

# Check negotiation cycle
condor_userprio -negotiator

# Verify submit file
condor_submit -dry-run job.sub

# SSH to running job
condor_ssh_to_job <job_id>

# Fetch output early
condor_transfer_data <job_id>
