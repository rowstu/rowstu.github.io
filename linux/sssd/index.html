sssd configuration and troubleshooting notes for sudo

ipa sudorule-add --cmd "/bin/su - deployer" --hostcat all --users=username --no-passwd
ipa sudorule-add-option <rule-name> --sudooption '!authenticate'

/etc/nsswitch.conf

sudoers: files sss

check sudo rule is applied

sudo -l

if not, flush the cache

# sss_cache -E

sssd.conf

[sssd]
services = nss, pam, sudo, ssh

[domain/your_idm_domain]
sudo_provider = ipa

check the rule

sudo su - deployer

more checks

sudo sssctl cache-expire
sudo sssctl domain-list
sudo sssctl user-checks username

add debug to logging sssd.conf

[domain/your_idm_domain]
debug_level = 9
