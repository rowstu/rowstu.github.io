Bubble Sort

O(n²) avg · O(n) best

Repeatedly swaps adjacent elements if they're in the wrong order. Simple but slow. Stable sort.

Insertion Sort

O(n²) avg · O(n) best

Builds the sorted array one element at a time by inserting each into its correct position. Efficient for small or nearly-sorted data. Stable.

Selection Sort

O(n²) avg · O(n²) best

Finds the minimum element and moves it to the front, repeatedly. Always O(n²) comparisons. Not stable.

Merge Sort

O(n log n) always

Divides the array in half, sorts each half, then merges them. Guaranteed O(n log n) but uses extra memory. Stable.

Quick Sort

O(n log n) avg · O(n²) worst

Picks a pivot, partitions around it, sorts each side. Very fast in practice. Not stable. Worst case with already-sorted data.

Heap Sort

O(n log n) always

Builds a max-heap, then repeatedly extracts the maximum. Guaranteed O(n log n) in all cases. In-place but not stable. Good worst-case guarantee.

Shell Sort

O(n log² n) avg · O(n²) worst

Generalises insertion sort by comparing elements far apart, reducing the gap until 1. Much faster than insertion sort for larger arrays. Not stable.
