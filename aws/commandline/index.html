1. Basic AWS CLI Setup

1.1 Installation Steps (Linux)

Debian/Ubuntu

sudo apt update
sudo apt install awscli -y

RHEL/CentOS

sudo yum install awscli -y

Option B: Install via Official AWS Installer (Latest Version)

curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install

Verify Installation

aws --version

1.2 Configure AWS Credentials

Run:

aws configure

Then enter:

AWS Access Key ID [None]: YOUR_ACCESS_KEY
AWS Secret Access Key [None]: YOUR_SECRET_KEY
Default region name [None]: us-east-1
Default output format [None]: json

1.3 Where Credentials Are Stored

AWS stores credentials at:

~/.aws/credentials
~/.aws/config

Example:

[default]
aws_access_key_id = YOUR_ACCESS_KEY
aws_secret_access_key = YOUR_SECRET_KEY
region = us-east-1
output = json

2. Common AWS CLI Commands

✅ S3 (Storage)

List Buckets

aws s3 ls

Upload a File

aws s3 cp myfile.txt s3://my-bucket/

Download a File

aws s3 cp s3://my-bucket/myfile.txt .

Sync Local Folder to S3

aws s3 sync ./local-folder s3://my-bucket/

✅ EC2 (Virtual Machines)

List Instances

aws ec2 describe-instances

Launch a New Instance

aws ec2 run-instances \
  --image-id ami-0abcdef1234567890 \
  --count 1 \
  --instance-type t3.micro \
  --key-name my-keypair \
  --security-group-ids sg-0123456789abcdef0 \
  --subnet-id subnet-0123456789abcdef0

Stop an Instance

aws ec2 stop-instances --instance-ids i-0123456789abcdef0

Start an Instance

aws ec2 start-instances --instance-ids i-0123456789abcdef0

✅ IAM (Identity & Access Management)

List Users

aws iam list-users

Create a Role

aws iam create-role \
  --role-name MyRole \
  --assume-role-policy-document file://trust-policy.json

Attach Policy to Role

aws iam attach-role-policy \
  --role-name MyRole \
  --policy-arn arn:aws:iam::aws:policy/AmazonS3ReadOnlyAccess

✅ Lambda (Serverless Functions)

List Functions

aws lambda list-functions

Invoke Function

aws lambda invoke \
  --function-name my-function \
  --payload '{"key": "value"}' \
  output.json

3. Procedures (Step-by-Step)

3.1 Launch an EC2 Instance

Step 1: Choose AMI

Get the AMI ID:

aws ec2 describe-images --owners amazon --filters "Name=name,Values=amzn2-ami-hvm-*-x86_64-gp2" --query "Images[0].ImageId"

Step 2: Create Key Pair

aws ec2 create-key-pair --key-name my-keypair --query 'KeyMaterial' --output text > my-keypair.pem
chmod 400 my-keypair.pem

Step 3: Create Security Group

aws ec2 create-security-group --group-name my-sg --description "SSH access"
aws ec2 authorize-security-group-ingress --group-name my-sg --protocol tcp --port 22 --cidr 0.0.0.0/0

Step 4: Launch Instance

aws ec2 run-instances \
  --image-id ami-0abcdef1234567890 \
  --count 1 \
  --instance-type t3.micro \
  --key-name my-keypair \
  --security-group-ids sg-0123456789abcdef0

3.2 Upload Files to S3

Create Bucket

aws s3 mb s3://my-bucket

Upload Folder

aws s3 sync ./myfolder s3://my-bucket/

3.3 Manage IAM Roles

Create Role

Create trust-policy.json:

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": { "Service": "ec2.amazonaws.com" },
      "Action": "sts:AssumeRole"
    }
  ]
}

Run:

aws iam create-role --role-name EC2Role --assume-role-policy-document file://trust-policy.json

Attach Policy

aws iam attach-role-policy --role-name EC2Role --policy-arn arn:aws:iam::aws:policy/AmazonS3ReadOnlyAccess

4. Advanced Techniques

4.1 Scripting AWS CLI Tasks

Example: Automated EC2 Backup Script

#!/bin/bash
INSTANCE_ID="i-0123456789abcdef0"
DATE=$(date +%F)

aws ec2 create-snapshot \
  --volume-id $(aws ec2 describe-instances --instance-ids $INSTANCE_ID --query "Reservations[0].Instances[0].BlockDeviceMappings[0].Ebs.VolumeId" --output text) \
  --description "Backup $DATE"

4.2 Monitoring & Logging (CLI)

CloudWatch Logs

List log groups:

aws logs describe-log-groups

Get latest logs:

aws logs get-log-events \
  --log-group-name /aws/lambda/my-function \
  --log-stream-name my-stream

CloudWatch Metrics

List metrics:

aws cloudwatch list-metrics --namespace AWS/EC2

5. Best Practices (Command Line Efficiency + Security)

5.1 Security Best Practices

  • Never store long-term access keys in your laptop.
  • Use IAM Roles for EC2, ECS, Lambda.
  • Use MFA and temporary session tokens (STS).
  • Use AWS Vault or SSO for safer credential management.
  • Use least privilege policies.

5.2 Efficiency Tips

  • Use JSON query filters:
aws ec2 describe-instances --query 'Reservations[*].Instances[*].InstanceId'
  • Use output formatting:
aws s3 ls --output table
  • Use profile switching:
aws configure --profile prod
aws s3 ls --profile prod

5.3 Resource Management

  • Tag everything:
aws ec2 create-tags --resources i-1234567890abcdef0 --tags Key=Name,Value=prod-web
  • Set up lifecycle policies for S3.
  • Use CloudWatch alarms for cost control.
