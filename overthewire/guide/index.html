Bandit: Linux Command Line Fundamentals

Bandit is the ideal starting point, teaching essential Linux skills through 33+ progressive levels. Each level reveals a password needed for the next, reinforcing the concept that access builds on knowledge.

Core Concepts Covered

1. File Navigation and Reading

The earliest levels teach fundamental file operations:

  • Hidden files: Files beginning with . are hidden from ls by default
  • Special characters in filenames: Filenames with spaces, dashes, or other special characters require careful handling
  • File type identification: The file command reveals what a file actually contains, regardless of extension

Key Commands:

# Reading files with special names
cat ./- filename        # Dash prefix
cat "spaces in name"    # Quoted spaces
cat spaces\ in\ name    # Escaped spaces

# Finding hidden files
ls -la                  # Show all including hidden

# Identify file type
file mysteryfile

2. File Searching

Finding the right file among many is a critical skill:

The find command - Your Swiss Army knife for file location:

# Find by name pattern
find /path -name "*.txt"

# Find by size (c=bytes, k=kilobytes, M=megabytes)
find . -size 1033c

# Find by ownership
find / -user username -group groupname

# Find by type (f=file, d=directory)
find . -type f

# Combine with execution
find . -type f -exec file {} +

Combining criteria - Real challenges often require multiple filters:

# Find human-readable, specific size, non-executable files
find . -type f -size 1033c ! -executable -exec file {} + | grep ASCII

3. Text Processing

Extracting specific information from files:

# Search within files
grep "pattern" filename
grep -r "pattern" directory/    # Recursive

# Extract unique lines
sort filename | uniq -u         # Only lines appearing once

# Extract strings from binary files
strings binaryfile | grep pattern

# Text transformation
cat file | tr '[A-Za-z]' '[N-ZA-Mn-za-m]'   # ROT13

4. Encoding and Compression

Data is often transformed or compressed:

Base64:

# Decode
echo "encoded==" | base64 -d
cat file | base64 -d

# Encode
echo "plaintext" | base64

Compression layers - Files may be compressed multiple times:

# Identify compression type
file compressed_file

# Decompress based on type
gzip -d file.gz
gunzip file.gz
bunzip2 file.bz2
tar -xvf archive.tar

# Hexdump reversal
xxd -r hexdump.txt > binary

Lesson Learned: When facing nested compression, work methodically - use file after each extraction to identify the next layer.

5. SSH Key Authentication

Moving beyond passwords:

# Connect using a private key
ssh -i private.key user@host

# Execute command remotely without interactive shell
ssh -i key user@host "command to run"

# Proper key permissions (required)
chmod 600 private.key

6. Network Connections

Communicating with network services:

# Basic connection with netcat
echo "data" | nc hostname port
nc -l port                      # Listen mode

# SSL/TLS connections
openssl s_client -connect host:port -quiet

# SSL with proper handling
openssl s_client -connect host:port -ign_eof

Port scanning for service discovery:

nmap -p 31000-32000 localhost

7. Privilege Escalation Concepts

Understanding how permissions work:

  • setuid binaries: Programs that run with the owner’s permissions, not the caller’s
  • Reading cron jobs: Scheduled tasks can reveal automated actions
  • Analyzing shell scripts: Understanding what scripts do before they’re executed
# Check file permissions
ls -la filename

# Find setuid binaries
find / -perm -4000 2>/dev/null

# View cron configurations
cat /etc/cron.d/*

8. Git Repository Exploration

Later levels introduce version control concepts:

# Clone via SSH with custom port
git clone ssh://user@host:port/path/to/repo

# Explore commit history
git log
git log --all --oneline

# Check all branches
git branch -a
git branch -r

# View tags (often overlooked!)
git tag
git show tagname

# See changes
git diff
git show commit_hash

Lesson Learned: Secrets often hide in commit history, tags, or branches - not just the current working tree.

9. Shell Escaping

Breaking out of restricted environments:

  • more and less: Press v to enter visual/vim mode
  • vim escape: :set shell=/bin/bash then :shell
  • Environment manipulation: Understanding how shells are configured

10. Multiplexing with tmux

When you need multiple terminal sessions:

# Start new session
tmux new -s sessionname

# Detach: Ctrl+b d

# Reattach
tmux attach -t sessionname

Krypton: Cryptography Fundamentals

Krypton teaches classical cryptography through hands-on cipher breaking. The progression mirrors the historical development of encryption.

Cryptographic Concepts Covered

1. Base64 Encoding

Understanding: Base64 is encoding, not encryption - it provides no security, just a way to represent binary data as ASCII text.

echo "encoded==" | base64 -d

2. Caesar/ROT Ciphers

The simplest substitution cipher shifts each letter by a fixed amount:

Plain:   A B C D E F G H I J K L M ...
Cipher:  D E F G H I J K L M N O P ...  (shift of 3)

Breaking approach:

  • If you can encrypt chosen plaintext, encrypt the alphabet to reveal the shift
  • Count the shift distance between plain and cipher alphabets

Using tr for rotation:

# ROT13 (shift 13)
cat ciphertext | tr '[A-Za-z]' '[N-ZA-Mn-za-m]'

# ROT12 (shift 12)
cat ciphertext | tr '[M-ZA-Lm-za-l]' '[A-Za-z]'

Python approach:

def caesar_shift(text, shift):
    result = ""
    for char in text:
        if char.isalpha():
            base = ord('A') if char.isupper() else ord('a')
            result += chr((ord(char) - base + shift) % 26 + base)
        else:
            result += char
    return result

3. Known Plaintext Attacks

When you can encrypt arbitrary text, you can determine the key:

  1. Create a file containing ABCDEFGHIJKLMNOPQRSTUVWXYZ
  2. Encrypt it using the target system
  3. Compare output to input to derive the mapping

4. Frequency Analysis

For substitution ciphers without the encryption oracle, analyze letter frequencies:

English letter frequency (approximate):

E T A O I N S H R D L C U M W F G Y P B V K J X Q Z

Python frequency counter:

from collections import Counter
import sys

with open(sys.argv[1]) as f:
    text = f.read()
    counter = Counter(c for c in text.upper() if c.isalpha())
    for letter, count in counter.most_common():
        print(f"{letter}: {count}")

Breaking process:

  1. Count letter frequencies in the ciphertext
  2. Map most frequent cipher letters to E, T, A, etc.
  3. Look for common patterns:
    • Single-letter words are likely A or I
    • Common three-letter words: THE, AND, FOR
    • Double letters: LL, SS, EE, OO
  4. Iteratively refine based on word patterns

Resources for frequency analysis:

  • The Black Chamber - Simon Singh’s cryptography resource
  • Notre Dame Cryptography Hints

5. Building Decryption Tools

Creating a substitution cipher translator:

message = "CIPHERTEXT HERE"

# Mapping derived from frequency analysis
plain =  "abcdefghijklmnopqrstuvwxyz"
cipher = "qwertyuiopasdfghjklzxcvbnm"  # Derived mapping

translated = ""
for char in message:
    if char.upper() in cipher.upper():
        index = cipher.upper().find(char.upper())
        translated += plain[index].upper() if char.isupper() else plain[index]
    else:
        translated += char

print(translated)

Key Lessons from Krypton

  1. Encoding != Encryption - Base64 provides no security
  2. Key reuse is dangerous - Same key for multiple messages allows comparison attacks
  3. Known plaintext is powerful - Any encryption system that lets you encrypt chosen text is vulnerable to key extraction
  4. Language has patterns - Statistical properties of language undermine simple substitution
  5. More ciphertext = easier analysis - The more encrypted text you have, the clearer the patterns

Natas: Web Application Security

Natas teaches web security through vulnerable web applications. Each level presents a different class of vulnerability.

Web Security Concepts Covered

1. Information Disclosure

The first lessons: not all security is about complex exploits.

View Page Source: Right-click → View Source, or Ctrl+U

  • Comments in HTML often contain sensitive information
  • Hidden form fields may reveal secrets
  • Debugging information left in production

Developer Tools: F12 in most browsers

  • Network tab shows all requests and responses
  • Application tab shows cookies, local storage
  • Elements tab shows current DOM state

2. HTTP Fundamentals

Understanding how web communication works:

Request components:

  • Method (GET, POST, PUT, DELETE)
  • Headers (authentication, cookies, referer)
  • Body (form data, JSON)

Response components:

  • Status codes (200, 301, 403, 404, 500)
  • Headers (cookies, content-type)
  • Body (HTML, JSON, binary)

Python requests library pattern:

import requests

url = 'http://target.site/'
auth = ('username', 'password')  # Basic auth

# GET request
response = requests.get(url, auth=auth)

# POST request with form data
response = requests.post(url, auth=auth, data={'field': 'value'})

# Custom headers
headers = {'Referer': 'http://expected.site/'}
response = requests.get(url, auth=auth, headers=headers)

3. Cookie Manipulation

Cookies store client-side state that can often be manipulated:

# Setting cookies with requests
cookies = {'loggedin': '1', 'admin': 'true'}
response = requests.get(url, auth=auth, cookies=cookies)

# Reading cookies from response
print(response.cookies)

Common vulnerabilities:

  • Boolean flags (loggedin=0 → loggedin=1)
  • Role indicators (role=user → role=admin)
  • Insecure encoding of values

4. XOR Encryption Weaknesses

XOR encryption is symmetric - the same operation encrypts and decrypts:

plaintext XOR key = ciphertext
ciphertext XOR key = plaintext
plaintext XOR ciphertext = key  // Key recovery!

Breaking XOR encryption:

def xor_strings(s1, s2):
    return ''.join(chr(ord(a) ^ ord(b)) for a, b in zip(s1, s2))

# If you know plaintext and ciphertext:
known_plain = '{"data":"known"}'
ciphertext = base64.b64decode(cookie)
key = xor_strings(known_plain, ciphertext)
# Key often repeats - look for patterns

5. Command Injection

When user input is passed to system commands:

Vulnerable pattern:

$output = shell_exec("grep " . $_GET['search'] . " dictionary.txt");

Exploitation:

Input: ; cat /etc/passwd
Result: grep ; cat /etc/passwd dictionary.txt
        (runs cat after grep fails)

Common injection characters:

  • ; - Command separator
  • | - Pipe output to next command
  • && - Run next if previous succeeds
  • || - Run next if previous fails
  • `command` - Command substitution
  • $(command) - Command substitution

Python automation:

# Inject command via POST parameter
payload = "; cat /etc/secret_file"
response = requests.post(url, auth=auth, data={"search": payload})

6. File Upload Vulnerabilities

When applications allow file uploads:

Extension bypass:

  • Application may check extensions but server executes differently
  • Try: .php, .php5, .phtml, .phar
  • Double extensions: shell.php.jpg

MIME type bypass:

  • Add magic bytes to fool file type detection
  • GIF header: GIF89a
  • PNG header: \x89PNG

Example malicious PHP:

<?php
GIF89a  // Bypass MIME check
system($_GET['c']);  // Command execution via parameter
?>

Lesson Learned: Never trust client-provided file information - always validate server-side.

7. Regex Bypass

When input filtering uses weak regular expressions:

Common bypasses:

  • Null bytes: file%00.txt (in older PHP)
  • Encoding variations: %2e%2e%2f for ../
  • Case sensitivity: CaT instead of cat
  • Alternative commands: head instead of cat

Path traversal:

../../../etc/passwd
....//....//....//etc/passwd  (if ../ is filtered once)

8. Authentication Weaknesses

HTTP Basic Auth: Credentials sent with every request (base64 encoded, not encrypted)

Session issues:

  • Predictable session IDs
  • Session fixation
  • Missing session invalidation

9. Regular Expression Tips for Extraction

Using Python regex to extract data from responses:

import re

content = response.text

# Find specific pattern
password = re.findall(r'password is (\w+)', content)[0]

# Find all matches
all_matches = re.findall(r'secret: (.+?)</div>', content)

# Case insensitive
matches = re.findall(r'key: (\w+)', content, re.IGNORECASE)

Key Lessons from Natas

  1. Always check the source - HTML comments and hidden fields reveal information
  2. Trust nothing from the client - Cookies, headers, and form data can all be manipulated
  3. Validate and sanitize input - Never pass user input to system functions
  4. Defense in depth - One security measure is never enough
  5. Automate your testing - Scripts let you iterate quickly and consistently

Problem-Solving Approach

General Methodology

  1. Read the challenge description carefully - Hints are embedded
  2. Enumerate - What do you have access to? What can you see?
  3. Research - Look up unfamiliar commands or concepts
  4. Hypothesize - Form theories about what might work
  5. Test - Try your theories, observe results
  6. Iterate - Refine based on feedback
  7. Document - Record what worked and why

When You’re Stuck

  1. Take a break - Fresh eyes find new solutions
  2. Revisit the basics - Did you miss something obvious?
  3. Check man pages - man command for detailed options
  4. Read error messages - They often point to the problem
  5. Simplify - Can you solve a smaller version of the problem?
  6. Research the concept - Not the specific challenge

Building Your Skills

  • Practice regularly - Security skills atrophy without use
  • Learn the fundamentals - Deep understanding beats memorized solutions
  • Build a toolbox - Create scripts for common operations
  • Join communities - Discord servers, forums, CTF teams
  • Try other wargames - Each platform teaches different things

Final Thoughts

The true value of OverTheWire isn’t in collecting passwords - it’s in developing a security mindset. Each challenge teaches you to:

  • Question assumptions - Just because something looks secure doesn’t mean it is
  • Think like an attacker - Understanding offense improves defense
  • Value persistence - Complex problems yield to sustained effort
  • Appreciate simplicity - The best security is often the most straightforward

Whether you’re pursuing a career in security, strengthening your development practices, or simply enjoying puzzles, these skills transfer broadly. The command-line fluency from Bandit improves daily work. The cryptographic intuition from Krypton helps evaluate security claims. The web security awareness from Natas makes you a better developer.

Keep learning, keep practicing, and remember: every expert was once a beginner staring at a terminal wondering what to type next.
