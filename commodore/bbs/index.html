Connecting Your C64 to BBSes

Home / Browse/ Commodorebbs

Bulletin Board Systems are still running in 2024, and connecting from a real C64 is one of the most satisfying retro computing experiences. Most modern BBSes accept telnet connections, and a cheap WiFi modem makes this trivially easy.

What You Need

  1. A C64 (any model)
  2. A WiFi modem â€” an ESP8266 or ESP32-based device that connects to the C64’s user port and provides a Hayes AT command interface over WiFi
  3. A terminal program on the C64
  4. Your WiFi network credentials

WiFi Modems

These are small boards that plug into the C64 user port and emulate a standard Hayes-compatible modem. Internally they connect to your WiFi and translate AT commands into TCP/telnet connections.

Terminal Programs

You need software on the C64 to act as a terminal. The modem handles the network connection; the terminal program handles the display, keyboard input, and ANSI/PETSCII translation.

ProgramNotes
CCGMSThe gold standard. Supports PETSCII, ASCII, and ANSI. Works with WiFi modems out of the box. Actively maintained

CCGMS is the recommended starting point. Download it from csdb.dk and load it onto your SD2IEC/Pi1541.

Setup â€” Step by Step

1. Configure the WiFi Modem

Power on the C64 with the modem attached. Load your terminal program.

Most modems start in command mode. Type these AT commands:

AT                          Test â€” should respond "OK"
AT+CONFIG                   Enter configuration mode (varies by modem)
AT$SSID=YourNetworkName     Set WiFi SSID
AT$PASS=YourPassword        Set WiFi password
AT$SAVE                     Save settings
ATC1                        Connect to WiFi

The exact commands vary by modem â€” check your modem’s documentation. Most have a one-time setup process.

2. Set Baud Rate

The C64 user port serial is typically set to 2400 baud by default. Most WiFi modems can go faster.

AT$BAUD=2400      Set to 2400 baud (safe, always works)
AT$BAUD=9600      Set to 9600 baud (faster, works with most terminal programs)
AT$BAUD=19200     Set to 19200 baud (pushing it â€” may get errors)

2400 baud is the safest choice. The C64’s user port UART is software-driven (bit-banged by the CIA chip), so higher baud rates depend on the terminal software handling interrupts fast enough.

CCGMS supports up to 9600 baud reliably with its custom driver.

3. Connect to a BBS

Once WiFi is connected, dial a BBS using the telnet AT command:

ATDT bbs.address.com:port

For example:

ATDT borderlinebbs.dyndns.org:6400
ATDT particlesbbs.dyndns.org:6400
ATDT blueboxbbs.ddns.net

The modem connects via telnet and you’re in.

4. Choosing PETSCII vs ANSI

BBSes typically support two display modes:

  • PETSCII â€” Native C64 character set. Looks best on real hardware. The BBS sends C64 screen codes directly. Choose this when available
  • ANSI â€” PC-style terminal graphics. CCGMS can translate ANSI escape codes to the C64 screen, but it’s not as clean as native PETSCII

When a BBS asks for your terminal type, choose Commodore 64 or PETSCII if offered.

Testing Your Connection

A good first test is to connect to a well-known BBS:

ATDT borderlinebbs.dyndns.org:6400

If you see the BBS login screen in PETSCII graphics, everything is working.

You can run a BBS on a real C64 or on an emulator.

You’ll need: a C64, storage (1541/SD2IEC), a modem or telnet bridge, and a lot of patience.

On an Emulator (Serving via Telnet)

Run VICE (C64 emulator) with a BBS program, and use tcpser to bridge telnet connections to the emulated modem:

# Install tcpser (translates telnet to modem commands)
# Then run VICE with the RS-232 port mapped to tcpser

tcpser -v 25232 -p 6400 -s 2400 -l 7
# -v 25232 = VICE connects here
# -p 6400  = telnet callers connect here
# -s 2400  = baud rate
# -l 7     = log level

This lets people telnet to your machine on port 6400 and appear as modem callers to the BBS software.

Tips

  • 2400 baud is fine. BBSes are text-based; speed isn’t critical and reliability matters more
  • CCGMS + any WiFi modem is the combo that works for almost everyone
  • Keep a list of BBS addresses â€” type them into your terminal program’s phonebook/dialling directory if it supports one
  • Log your sessions â€” CCGMS can capture text to disk if you want to save messages
  • Join the BBS community â€” BBSes are social. Post messages, join discussions, play door games. It’s a different pace from modern internet and that’s the point
