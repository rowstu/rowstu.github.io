File Transfer and Disk Images on the C64

Home / Browse/ Commodorefile Transfer

Getting software onto a real C64 from a modern PC is a common task. This covers the file formats, tools, and workflows for moving data between the two worlds.

File Formats

FormatExtensionWhat it is
D64.d64Disk image of a 1541 floppy (170KB, 35 tracks). The most common format
D71.d71Disk image of a 1571 floppy (340KB, double-sided)
D81.d81Disk image of a 1581 floppy (800KB, 3.5" disk)
G64.g64Low-level GCR disk image. Preserves copy protection and non-standard tracks
PRG.prgA single C64 program file. First two bytes are the load address
CRT.crtCartridge ROM image. Used by cartridge-port devices (EasyFlash, Kung Fu Flash)
T64.t64Tape image. Emulates a datasette recording
TAP.tapRaw tape image with pulse timing data. More accurate than T64
SID.sidSID music file. Playable in SID players (HVSC format)
SEQ.seqSequential file (text/data)
REL.relRelative file (random access)

For day-to-day use, you’ll mostly deal with D64 and PRG files.

PC-Side Tools

DirMaster (Windows)

The best tool for working with D64/D71/D81 images on Windows.

  • Browse disk images like a file manager
  • Extract PRG files from D64 images
  • Create new blank disk images
  • Import files into disk images
  • View PETSCII directory art
  • Free download from style64.org

c1541 (Cross-Platform)

Command-line tool included with the VICE emulator. Works on Linux, macOS, Windows.

# List contents of a disk image
c1541 game.d64 -list

# Extract a file
c1541 game.d64 -read "filename" output.prg

# Create a new blank disk image
c1541 -format "my disk,01" d64 newdisk.d64

# Write a PRG into a disk image
c1541 newdisk.d64 -write myprogram.prg "myprogram"

# Delete a file from a disk image
c1541 game.d64 -delete "filename"

# Copy files between disk images
c1541 source.d64 -read "file" temp.prg
c1541 dest.d64 -write temp.prg "file"

cc65 / ACME / KickAssembler

If you’re writing assembly, these cross-assemblers run on modern PCs and output PRG files you can transfer to the C64.

# ACME assembler example
acme -f cbm -o game.prg game.asm

# Then write to a D64
c1541 disk.d64 -write game.prg "game"

Getting Files to the C64

Via SD2IEC

The simplest method. The SD2IEC reads SD cards directly.

  1. Format an SD card as FAT32 on your PC
  2. Copy D64/D81/PRG files to the SD card
  3. Insert the card into the SD2IEC
  4. On the C64:
    • LOAD"$",8 then LIST to see the SD card contents
    • Navigate into D64 images with LOAD"imagename.d64",8
    • Load PRG files directly with LOAD"filename.prg",8,1

With JiffyDOS: @$ to list, /filename to load.

Folder structure: SD2IEC supports subdirectories. Organise your SD card into folders:

/games/
  /arcade/
    galaga.d64
    donkeykong.d64
  /rpg/
    ultima4.d64
/demos/
/tools/

Navigate with CD:foldername or LOAD"CD:foldername",8.

Via Pi1541

Same as SD2IEC but uses an SD card in the Raspberry Pi. Supports D64, G64, and PRG formats. Has an OLED display for browsing.

Via 1541 Ultimate II+

  • Built-in file browser on the C64 screen
  • USB port — plug in a USB stick
  • Ethernet — FTP server for wireless file transfer
  • Can mount D64/G64/D71/D81 images as virtual drives
  • REU files can be loaded automatically

The FTP feature is particularly nice — ftp 192.168.1.x from your PC and drag files across.

Via Ethernet (1541 Ultimate)

# From PC, connect via FTP
ftp 192.168.1.100
# Default: no username/password
# Navigate to the SD card and upload files
put game.d64

Writing Real Floppies (ZoomFloppy)

If you want to use actual 5.25" disks with a PC:

  1. Get a ZoomFloppy adapter (USB-to-IEC)
  2. Connect a real 1541 drive to the ZoomFloppy, and the ZoomFloppy to your PC via USB
  3. Use OpenCBM software:
# Install OpenCBM (Linux)
sudo apt install opencbm

# List directory of the disk in the drive
cbmctrl dir 8

# Write a D64 image to a real floppy
d64copy image.d64 8

# Read a real floppy to a D64 image
d64copy 8 backup.d64

# Transfer a single file
cbmcopy -r 8 "filename"

Emulators

For testing and development, emulators are indispensable.

EmulatorPlatformNotes
VICELinux, macOS, WindowsThe reference emulator. Cycle-exact. Includes C64, C128, VIC-20, Plus/4, PET
FrodoMulti-platformLightweight, not cycle-exact but fast
Hoxs64WindowsExcellent cycle-exact emulation, good for demos
micro64WebBrowser-based, quick testing

VICE Quick Start

# Install on Linux
sudo apt install vice

# Run with a disk image
x64sc game.d64

# Run a PRG directly
x64sc -autostart game.prg

# x64sc is the cycle-exact version (preferred)
# x64 is the older, faster but less accurate version

VICE keyboard mapping: Use F12 for the menu. The RESTORE key is mapped to Page Up. RUN/STOP is mapped to Escape.

  • Gamebase64 (gb64.com) — Comprehensive game database with downloads
  • HVSC (hvsc.c64.org) — High Voltage SID Collection. 50,000+ SID music files
  • Lemon64 (lemon64.com) — Community forums, game reviews, screenshots
  • Commodore.software — Curated archive of C64/C128/Amiga software
  • Archive.org — Massive collection of disk images and tape images
