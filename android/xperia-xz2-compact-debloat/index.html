Sony Xperia XZ2 Compact - Stock Firmware Debloat

Conservative, banking-app-safe debloat guide for the Sony Xperia XZ2 Compact (xz2c) on stock firmware with locked bootloader.

Key Principles

  • Do not unlock the bootloader if banking apps are a hard requirement. Play Integrity / hardware attestation will fail on an unlocked bootloader with a custom ROM.
  • Use pm uninstall --user 0 instead of deleting system APKs. This keeps you recoverable and avoids bootloops.
  • Disable before uninstall for anything you’re unsure about. Test for a day, then uninstall if stable.
  • Reboot and test after every batch - check calls, SMS, camera, Wi-Fi, Bluetooth, fingerprint, Play Store updates, and banking apps.

Prerequisites

Enable Developer options > USB debugging on the device.

adb devices
adb shell getprop ro.build.version.release

Make an Inventory First

Always snapshot your current state before making changes:

# List all packages
adb shell pm list packages > packages_all.txt

# List Sony-specific packages
adb shell pm list packages | grep -i sony > packages_sony.txt
adb shell pm list packages | grep -i semc >> packages_sony.txt
adb shell pm list packages | grep -i xperia >> packages_sony.txt

# List installed for user 0 (useful for rollback)
adb shell cmd package list packages --user 0 > packages_user0.txt

Core ADB Commands

Remove an app (recommended method)

adb shell pm uninstall --user 0 <package.name>

Disable an app (safer for uncertain packages)

adb shell pm disable-user --user 0 <package.name>

Restore a removed app

adb shell cmd package install-existing <package.name>

Re-enable a disabled app

adb shell pm enable <package.name>

Investigate an unknown package

adb shell cmd package resolve-activity --brief <package.name>
adb shell dumpsys package <package.name> | head -n 60

Finding Bloatware

# Third-party bloat
adb shell pm list packages | grep -i -E "facebook|meta|netflix|amazon|booking|spotify|opera|tiktok|linkedin|games"

# Promo / trial / support junk
adb shell pm list packages | grep -i -E "promo|trial|recommend|tips|demo"

# Sony-specific
adb shell pm list packages | grep -i -E "sony|xperia|semc"

# Google apps
adb shell pm list packages | grep -i google

# Hidden AVG antivirus (Sony bundles this as com.s.antivirus)
adb shell cmd package list packages -f | grep -i antivirus

Enthusiast-Grade Optimisation

Kill Hidden Background Drains

# Stop apps waking the phone constantly
adb shell cmd appops set com.google.android.googlequicksearchbox RUN_IN_BACKGROUND deny
adb shell cmd appops set com.google.android.apps.wellbeing RUN_IN_BACKGROUND deny
adb shell cmd appops set com.google.android.feedback RUN_IN_BACKGROUND deny
adb shell cmd appops set com.sonymobile.smartnotification RUN_IN_BACKGROUND deny
adb shell cmd appops set com.sonymobile.prediction RUN_IN_BACKGROUND deny

# Disable analytics wakeups
adb shell cmd appops set com.sonymobile.anondata WAKE_LOCK deny
adb shell cmd appops set com.sonyericsson.idd.agent WAKE_LOCK deny
adb shell cmd appops set com.sonymobile.gotaidd.service WAKE_LOCK deny

Aggressive Battery Optimisation

# Force deep doze properly

Reduce Google Telemetry

Lock Background Location Abuse

adb shell cmd appops set --all ACCESS_BACKGROUND_LOCATION ignore

Verify Wakelock Sanity

After a few hours idle:

adb shell dumpsys batterystats | grep Wake

Post-OTA Notes

Sony OTAs refresh /system, /product, and /vendor. They often re-register system packages for user 0, so removed bloatware may reappear. This is normal.

Run the debloat script again after each OTA update.

Post-Debloat Verification

After reboot:

  1. Open Play Store, let it update Google Play Services
  2. Confirm fingerprint unlock works
  3. Test banking apps
  4. Verify calls + SMS work
  5. Check camera and Wi-Fi

Snapshot your clean state:

adb shell pm list packages --user 0 > clean_state.txt
