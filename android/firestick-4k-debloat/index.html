Amazon Fire Stick 4K - Debloat, Remove Telemetry & Advertising

Strip Amazon telemetry, tracking, advertising, and bloatware from a Fire Stick 4K over ADB on port 5555. No root required. All changes are recoverable.

Key Principles

  • Fire OS is a heavily modified Android fork. Amazon layers advertising, telemetry, and shopping on top of the OS.
  • ADB connects over the network on port 5555 (not USB like phones).
  • Use pm uninstall -k --user 0 to remove apps. The -k flag keeps data/cache.
  • Use pm disable-user --user 0 for system-critical packages you want to silence but not remove.
  • Everything is recoverable via cmd package install-existing or pm enable.
  • Do NOT remove settings or launcher core - you will lose the ability to navigate the device.

Prerequisites

Enable ADB on the Fire Stick

  1. Settings > My Fire TV > Developer Options
  2. Turn on ADB Debugging
  3. Note the Fire Stick’s IP address: Settings > My Fire TV > About > Network

Connect from your PC

adb connect <FIRESTICK_IP>:5555
adb devices

Accept the authorisation prompt on the TV screen.

Make an Inventory First

# All packages
adb shell pm list packages > firestick_packages_all.txt

# Amazon-specific
adb shell pm list packages | grep -i amazon > firestick_packages_amazon.txt

# Currently disabled
adb shell pm list packages -d > firestick_packages_disabled.txt

Core ADB Commands

Remove an app

adb shell pm uninstall -k --user 0 <package.name>

Disable an app

adb shell pm disable-user --user 0 <package.name>

Restore a removed app

adb shell cmd package install-existing <package.name>

Re-enable a disabled app

adb shell pm enable <package.name>

Disable Only - OTA Updates

Disable to prevent Amazon pushing updates that re-enable bloatware and may break ADB access. Be aware this means no security patches.

adb shell pm disable-user --user 0 com.amazon.device.software.ota
adb shell pm disable-user --user 0 com.amazon.device.software.ota.override
adb shell pm disable-user --user 0 com.amazon.settings.systemupdates
adb shell pm disable-user --user 0 com.amazon.tv.forcedotaupdater.v2

Disable Only - Security & Auth Services

These tie into the Amazon account/DRM flow. Disable rather than remove.

adb shell pm disable-user --user 0 com.amazon.securitysyncclient
adb shell pm disable-user --user 0 com.amazon.sharingservice.android.client.proxy

Disable Only - Parental Controls

Remove if you don’t need them, but disable is safer:

adb shell pm disable-user --user 0 com.amazon.parentalcontrols
adb shell pm disable-user --user 0 com.amazon.tv.parentalcontrols

DO NOT TOUCH - Red Line Packages

Removing these will brick navigation, prevent factory reset, or break core functionality:

  • com.amazon.tv.settings.core - Settings core (cannot navigate without this)
  • com.amazon.device.settings - Device settings
  • com.amazon.device.settings.sdk.internal.library - Settings SDK
  • com.amazon.tv.launcher - Only remove if you have an alternative launcher installed and set as default first
  • com.amazon.webview.chromium - System WebView (apps need this to render)
  • com.amazon.providers - Content providers
  • com.amazon.acos.providers.UnifiedSettingsProvider - Settings provider
  • com.amazon.identity.auth.device.authorization - Device auth (Amazon account)
  • com.amazon.platform.fdrw - Factory reset (removing = no recovery)

Optional - Disable Amazon Launcher Ads

If you want to replace the ad-heavy Fire TV home screen, install an alternative launcher first (e.g., Wolf Launcher, FLauncher from the Downloader app), then:

adb shell pm disable-user --user 0 com.amazon.tv.launcher
adb shell pm disable-user --user 0 com.amazon.firehomestarter

Important: Only do this AFTER confirming your alternative launcher works. If you lose the launcher with no replacement, you’ll need to factory reset.

Network-Level Blocking (Optional)

For additional privacy, block Amazon telemetry domains at your router or Pi-hole:

# Amazon telemetry / advertising domains
device-metrics-us.amazon.com
device-metrics-us-2.amazon.com
mads-adx.amazon.com
aax-us-east.amazon-adsystem.com
fls-na.amazon.com
unagi.amazon.com
unagi-na.amazon.com

Or set a Private DNS on the device via ADB:

adb shell settings put global private_dns_mode hostname
adb shell settings put global private_dns_specifier dns.adguard.com

Post-Debloat Notes

  • OTA updates may re-enable packages. If you disabled OTA, this won’t happen. If you allow updates, re-run the debloat script after each update.
  • Some packages may not exist on your specific Fire Stick model/firmware version. The script handles this gracefully (returns “not installed”).
  • Prime Video should still work after debloating - the core video player packages are not in the remove list. If it breaks, restore com.amazon.avod and com.amazon.venezia.
  • ADB access can be revoked by Amazon in firmware updates. Some newer FireOS versions (8.1.5.3+) have patched local ADB exploits. Keep ADB enabled and avoid updating if this is critical for you.
