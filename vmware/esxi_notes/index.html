VMware ESXi

Home / Browse/ Vmwareesxi_notes

VMware ESXi is a bare-metal hypervisor that installs directly on server hardware and enables you to run multiple virtual machines (VMs) on a single physical host.

Key Concepts

TermDefinition
HypervisorSoftware layer that enables virtualization. ESXi is a Type-1 hypervisor.
VMkernelThe core OS of ESXi that manages CPU, memory, storage, and networking for VMs.
VMFSVMware File System used for storing VM files.
vSphere ClientGUI/HTML5 interface for managing ESXi hosts and vCenter.
vCenter ServerCentral management platform for multiple ESXi hosts.
vMotionLive migration of VMs between ESXi hosts with no downtime.
HA (High Availability)Automatic restart of VMs on other hosts if a host fails.
DRS (Distributed Resource Scheduler)Balances workloads across hosts automatically.
vSANSoftware-defined storage that pools local storage across hosts.

Installation & Configuration (Virtual Environment)

Requirements

  • ESXi ISO

  • Minimum hardware (virtual):

    • 2 vCPUs
    • 4 GB RAM
    • 40 GB disk
    • 1 NIC

Step-by-Step Installation

  1. Power on hardware

  2. Choose Installer ISO

    • Select VMware ESXi ISO

  3. Guest OS

    • Select “Other” → “VMware ESXi”

  4. Allocate Resources

    • 2+ vCPUs
    • 4+ GB RAM
    • 40+ GB disk

  5. Network

    • Configure IP address

  6. Post-install Configuration

    • Set root password
    • Configure management network (IP, DNS, gateway)

Initial Setup Checklist

  • Verify network connectivity
  • Confirm DNS resolution
  • Configure NTP
  • Set syslog destination
  • Enable SSH for remote management (if needed)

Integration with VMware Products

ESXi + vSphere

ESXi is the compute layer of vSphere, while vSphere is the overall suite (including vCenter, DRS, HA, etc.)

Common vSphere Components:

  • vCenter Server
  • ESXi Hosts
  • vSphere Client
  • vSAN
  • NSX (Network virtualization)

ESXi + vSAN

  • vSAN aggregates local storage across multiple ESXi hosts.

  • Creates a shared datastore without external SAN.

  • Requires:

    • Minimum 3 hosts (for production)
    • Cache + Capacity disks per host
    • vSAN license

ESXi + vMotion

  • Enables live migration of VMs.

  • Requires:

    • Shared storage (NFS/VMFS/vSAN)
    • VMkernel network for vMotion
    • Compatible CPU families or EVC enabled

Troubleshooting & Best Practices

Common Issues

1. Host not reachable

  • Check management network
  • Confirm IP/DNS settings
  • Verify firewall rules

2. VM performance problems

  • Check CPU ready time
  • Check memory swap/ballooning
  • Monitor disk latency
  • Confirm network saturation

3. Storage problems

  • Verify datastore connectivity
  • Check path status
  • Confirm multipathing configuration
  • Validate datastore space

4. vMotion fails

  • Check VMkernel vMotion network
  • Confirm shared storage availability
  • Verify EVC compatibility

Best Practices

Hardware & Compatibility

  • Use VMware Compatibility Guide (HCL)
  • Use supported NICs and storage controllers

Security

  • Enable lockdown mode
  • Use strong root passwords
  • Keep ESXi patched
  • Disable unnecessary services

Networking

  • Use separate VLANs for:

    • Management
    • vMotion
    • Storage
    • VM traffic

  • Use 10GbE for vMotion and storage where possible

Storage

  • Use VMFS or vSAN for production
  • Ensure redundancy and backup strategy
  • Monitor datastore capacity

Backup & DR

  • Use VMware snapshots cautiously (not a backup)
  • Use Veeam/Commvault/Native tools for backups
  • Test restores regularly

Maintenance

  • Patch regularly using:

    • VMware Update Manager (VUM)
    • Lifecycle Manager

  • Use maintenance mode before upgrades
