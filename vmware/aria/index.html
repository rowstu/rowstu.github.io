üîó 2. Integration with vSphere, ESXi, and NSX

A. Prerequisites

Before integration:

  • Ensure network connectivity between Aria appliances and vCenter/NSX Manager (DNS, NTP/Time sync).
  • Administrative credentials for vCenter, NSX Manager.
  • Appropriate licenses for Aria products (Automation, Operations, Networks). (vmware.com)

B. Integrating Aria Operations with vSphere (vCenter / ESXi)

Steps

  1. Deploy Aria Operations appliances (cluster: master, replica, data nodes). (community.broadcom.com)

  2. In Aria Operations UI:

    • Navigate to Administration ‚Üí Solutions / Endpoints.
    • Add vCenter Server endpoint, supplying vCenter FQDN/IP and credentials. (techdocs.broadcom.com)

  3. Enable data collection (metrics, inventory, events).

  4. Verify ESXi hosts and VMs appear under inventory.

  5. Tag objects to correlate performance, compliance, and capacity.

Configuration details

  • Use collector groups/cloud proxies for distributed sites to central Aria Ops cluster. (community.broadcom.com)
  • Verify vSphere Distributed Switch (VDS) and datastores are visible.

C. Integrating Aria Operations with NSX

Steps

  1. From Aria Operations Solutions/Add Endpoint, select NSX Manager.
  2. Enter NSX Manager API endpoint and credentials.
  3. Allow Aria Ops to ingest topology, flows, distributed firewall metadata.
  4. Confirm NSX dashboards and object views in Aria Ops UI. (techdocs2-prod.adobecqms.net)

Notes

  • Full support for NSX Federation may vary depending on versions. (vmware.github.io)
  • For distributed traffic flow data, enable IPFIX/NetFlow exports on vSphere and NSX infrastructure. (vmware.com)

D. Integrating Aria Automation

Steps

  1. Add cloud accounts for vSphere and NSX in Aria Automation.

    • Cloud accounts let Automation create and manage vCenter and NSX objects. (Cloud Blogger)

  2. Create project and network zones tied to NSX segments/edge gateways.

  3. Define cloud templates that include vSphere compute and NSX network/security constructs. (Cloud Blogger)

  4. Test provisioning a VM with specific NSX networks/security groups.

Configuration essentials

  • Align NSX network resources (‚ÄúNSX_Network‚Äù resource type) in Automation inventory.
  • For NSX-based network automation, use Policy mode APIs where supported. (VMware Blogs)

üöÄ 3. Best Practices: Performance & Security

Performance Optimization

  • Segment Aria workloads: separate collectors and proxies for remote data centers. (community.broadcom.com)
  • Size Aria appliances according to environment scale (hosts, VMs, flows).
  • Use tagging and object relationships for efficient analytics queries.

Security Hardening

  • Use least privilege accounts for vCenter/NSX integration.
  • Terminate and validate certificates between Aria and endpoints (TLS mutual auth).
  • Configure role-based access control in Aria Automation and Operations.
  • Enable logging and integrate Aria Operations for Logs / SIEM ingestion.
