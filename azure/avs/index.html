2. Deployment Procedures

Prerequisites & Planning

Before deployment:

  1. Azure Subscription with the right quotas. (Microsoft Learn)
  2. Network planning including ExpressRoute/VNet topology and private IP blocks that do not overlap with on-premises networks. (Citrix Community)
  3. Minimum three hosts required to deploy an AVS private cloud. (Microsoft Azure)
  4. Portable VMware Cloud Foundation (VCF) license is now required (BYOL) for hosts. (Microsoft Azure)

Tip: Map IP ranges and network boundaries early to avoid overlay and peering issues.

üìå Recommended planning guide: AVS Planning and Deployment Guide (PDF)

Step-by-Step Deployment

  1. Register the AVS resource provider

    • In the Azure portal or Azure CLI: az provider register -n Microsoft.AVS
    • This ensures Azure can provision AVS. (Microsoft Learn)

  2. Create a resource group

    • Logical container for the private cloud.

  3. Create the AVS Private Cloud

    • Search Azure VMware Solution in the portal.
    • Select SKU (e.g., AV36, AV48) and cluster size (minimum 3 hosts). (Microsoft Learn)
    • Assign a non-overlapping /22 or larger management CIDR block.

  4. Network Connectivity

    • Deploy/attach ExpressRoute or VPN to connect AVS to Azure VNets and on-premises. (Microsoft Learn)
    • Validate routes and reachability.

  5. Validate Access

    • Access vCenter, NSX Manager via the portal.
    • Confirm network access according to policies.

‚è±Ô∏è Provisioning notes: Initial creation can take several hours; adding nodes later scales relatively faster. (Microsoft Learn)

Common Pitfalls to Avoid

  • CIDR overlaps with on-prem or Azure VNets ‚Äî leads to routing issues. (Citrix Community)
  • Insufficient quotas for hosts or ExpressRoute. Plan early.
  • Improper connectivity planning ‚Äî ExpressRoute or VPN misconfigurations.
  • Assuming native Azure VM IP models ‚Äî AVS uses VMware networking models.

3. Management and Operations

Management Tools

  • vCenter Server & vSphere Client for VM management (power, templates, performance). (azure-int.microsoft.com)
  • NSX Manager for network and firewall operations. (azure-int.microsoft.com)
  • VMware HCX for migrations and network extensions. (azure-int.microsoft.com)
  • Azure Portal/Azure CLI for private cloud lifecycle (create, scale, policies). (Microsoft Learn)
  • Azure Monitor/Azure Log Analytics for health monitoring.

üí° Azure Arc for AVS ‚Äî enables control and VM operations from Azure native tools. (Microsoft Learn)

Monitoring & Troubleshooting

Best practices:

  • Use Azure Monitor with diagnostics enabled for vCenter logs. (Microsoft Learn)
  • Set up Service Health alerts for maintenance notifications. (Microsoft Learn)
  • Define SOPs for health checks and incident remediation. (Microsoft Learn)
  • Track vSAN storage utilization (<75% recommended) ahead of maintenance windows. (Microsoft Learn)

Scaling and Optimization

  • Scale hosts or clusters as demand grows (private clouds can go up to 96 hosts). (TECHCOMMUNITY.MICROSOFT.COM)
  • Resize clusters during off-peak windows.
  • Use reserved instances to lower compute costs. (Microsoft Azure)
  • Monitor host performance and clusters via vSphere DRS.
  • Use Azure native services (e.g., Azure NetApp Files) to back high-performance storage tiers.

4. Security Considerations

Best Practices

  • Private connectivity with ExpressRoute or secured VPN; avoid public management endpoints. (Microsoft Learn)
  • NSX firewall rules to microsegment tenant networks.
  • Use Azure Firewall Premium to protect exposed services. (Microsoft Learn)
  • Enable Microsoft Defender for Cloud for hybrid threat protection. (Microsoft Learn)
  • Follow VMware and Azure security baselines. (Microsoft Learn)

Compliance & Governance

  • Use Azure Policy for role-based access and compliance guardrails.
  • Use Cost governance tools to define budgets and alerts. (Microsoft Learn)
  • Host workloads in regions meeting data residency requirements.
