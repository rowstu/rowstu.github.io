Cisco Switch Password Recovery

Complete guide to recovering access to a Cisco switch when passwords are unknown, including methods to decrypt stored passwords.

Password Recovery Process

Prerequisites

  • Physical console access - You must have direct console connection
  • Power control - Ability to power cycle the switch
  • Time - Allow 15-30 minutes for the process

Standard Password Recovery Procedure

Step 1: Connect to Console

Connect via console cable (9600 baud, 8-N-1).

Step 2: Power Cycle and Interrupt Boot

Method A: Power cycle

  1. Unplug power cable
  2. Wait 10 seconds
  3. Plug power back in
  4. Immediately hold the Mode button (front of switch) for 10-15 seconds

Method B: Interrupt boot sequence

  1. Power cycle the switch
  2. Press and hold Ctrl+Break (or Break key) within first 60 seconds of boot
  3. You should see ROMMON mode
switch:

Note: Timing is critical. If you miss it, power cycle and try again.

Step 3: Initialize Flash Filesystem

switch: flash_init
Initializing Flash...

Step 4: Rename Configuration File

This prevents the switch from loading the startup configuration:

switch: rename flash:config.text flash:config.old

Alternative names on some models:

  • flash:config.text
  • flash:startup-config
  • flash:config.cfg

To see available files:

switch: dir flash:

Step 5: Boot the Switch

switch: boot

The switch will boot without loading the configuration (no passwords).

Step 6: Skip Initial Configuration

Would you like to enter the initial configuration dialog? [yes/no]: no

Step 7: Enter Privileged Mode

Switch> enable
Switch#

No password required!

Step 8: Restore Configuration

Switch# rename flash:config.old flash:config.text

Step 9: Copy to Running Configuration

Switch# copy flash:config.text system:running-config

Or:

Switch# copy startup-config running-config

Step 10: Change the Password

Switch# configure terminal
Switch(config)# enable secret NewPassword123!
Switch(config)# exit

Step 11: Save Configuration

Switch# copy running-config startup-config
# Or
Switch# write memory

Step 12: Reload to Verify

Switch# reload

After reload, test the new password.

Decrypting Cisco Type 7 Passwords

Cisco Type 7 passwords are weakly encrypted and can be easily reversed.

Identifying Type 7 Passwords

In the configuration, look for:

username admin password 7 094F471A1A0A
enable password 7 06120A2E4348
line console 0
 password 7 05080F1C2243

The 7 indicates Type 7 encryption.

Decryption Methods

Method 1: Online Tools

Several websites can decrypt Type 7:

  • https://www.ifm.net.nz/cookbooks/cisco-ios-enable-secret-password-cracker.html
  • Search for “Cisco Type 7 decoder”

Warning: Don’t use online tools for sensitive production passwords.

Method 2: Python Script

#!/usr/bin/env python3

def decrypt_type7(encrypted):
    """Decrypt Cisco Type 7 password"""
    xlat = [
        0x64, 0x73, 0x66, 0x64, 0x3b, 0x6b, 0x66, 0x6f,
        0x41, 0x2c, 0x2e, 0x69, 0x79, 0x65, 0x77, 0x72,
        0x6b, 0x6c, 0x64, 0x4a, 0x4b, 0x44, 0x48, 0x53,
        0x55, 0x42
    ]

    index = int(encrypted[:2])
    encrypted = encrypted[2:]

    decrypted = ""
    for i in range(0, len(encrypted), 2):
        decrypted += chr(int(encrypted[i:i+2], 16) ^ xlat[index])
        index = (index + 1) % 26

    return decrypted

# Usage
encrypted_password = "094F471A1A0A"
print(f"Decrypted: {decrypt_type7(encrypted_password)}")

Method 3: Cisco IOS Command

If you have privileged access to another Cisco device:

Router# configure terminal
Router(config)# key chain test
Router(config-keychain)# key 1
Router(config-keychain-key)# key-string 7 094F471A1A0A
Router(config-keychain-key)# exit
Router(config-keychain)# exit
Router(config)# exit
Router# show key chain test
Key-chain test:
    key 1 -- text "decrypted_password"

Decrypting Cisco Type 5 Passwords

Type 5 passwords use MD5 hashing and are more secure.

Identifying Type 5 Passwords

enable secret 5 $1$mERr$hx5rVt7rPNoS4wqbXKX7m0
username admin secret 5 $1$GzA4$QN4zT5sKCy.8PLt7gLDHp1

The 5 indicates MD5 hash (Type 5).

Important Notes

  • Type 5 cannot be decrypted - it’s a one-way hash
  • Can only be cracked using brute force or dictionary attacks
  • Strength depends on password complexity

Cracking Type 5 (For Authorized Testing Only)

Using John the Ripper

# Create a file with the hash
echo 'admin:$1$mERr$hx5rVt7rPNoS4wqbXKX7m0' > hash.txt

# Run John the Ripper
john --format=md5crypt hash.txt

# Show cracked password
john --show hash.txt

Using Hashcat

# Type 500 = md5crypt
hashcat -m 500 -a 0 hash.txt wordlist.txt

# With rules
hashcat -m 500 -a 0 hash.txt wordlist.txt -r rules/best64.rule

Decrypting Cisco Type 15 Passwords (SCRYPT)

Type 15 passwords use SCRYPT hashing - very difficult to crack.

Identifying Type 15 Passwords

enable secret 15 $15$salt$8Ry2YdlnnNAuiEF6lxGq5dKG3h56Tn0

Cracking Type 15

  • Extremely difficult - designed to resist cracking
  • Very time/resource intensive
  • Best option: Use password recovery procedure instead

Alternative Recovery Methods

Method 1: Social Engineering / Documentation

  • Check password management systems
  • Ask previous administrators
  • Check documented procedures
  • Review change tickets

Method 2: TFTP Configuration Backup

If you have a recent backup on TFTP server:

Switch# copy tftp: flash:config.txt
Switch# copy flash:config.txt system:running-config

Then view passwords:

Switch# show running-config

Method 3: SNMP Community String

If SNMP is enabled with RW community:

# Change enable password via SNMP (if configured)
snmpset -v2c -c private 192.168.1.10 enterprises.9.2.1.1.1.3.1 s newpassword

Note: This rarely works on modern IOS versions.

Preventing Lockout

Use Password Vault

  • Store all passwords in secure password manager
  • Document password location
  • Maintain offline backup

Create Recovery Account

Switch(config)# username recovery privilege 15 secret RecoveryPass123!

Document Everything

Maintain documentation including:

  • Current passwords
  • Configuration backups
  • Recovery procedures
  • Contact information

Regular Backups

! Automated backup via script
Switch# copy running-config tftp://192.168.1.100/switch-backup.cfg

! Or via SCP (more secure)
Switch# copy running-config scp://user@192.168.1.100/backup/switch.cfg

Security Considerations

Password Recovery Security Risk

Password recovery procedure represents a physical security vulnerability:

  • Anyone with console access can reset passwords
  • No authentication required in ROMMON mode
  • Physical security is critical

Mitigations

1. Physical Security

  • Lock network closets
  • Restrict console access
  • Use security cameras
  • Implement access logs

2. Enable Password Recovery Disable

Warning: This prevents password recovery but also locks you out permanently if you forget the password!

Switch(config)# no service password-recovery

When enabled, you’ll see on boot:

PASSWORD RECOVERY FUNCTIONALITY IS DISABLED

To recover, you must erase the entire switch (factory reset).

3. Use Strong Passwords

! Type 5 (MD5) - good
Switch(config)# enable secret 5 ComplexP@ssw0rd!123

! Type 15 (SCRYPT) - better (newer IOS)
Switch(config)# enable algorithm-type scrypt secret VeryStr0ng!P@ssword#456

4. Multi-Factor Authentication

For newer IOS versions:

Switch(config)# aaa new-model
Switch(config)# aaa authentication login default group tacacs+ local
Switch(config)# tacacs-server host 192.168.1.50

Troubleshooting Password Recovery

Can’t Enter ROMMON Mode

Problem: Ctrl+Break doesn’t work

Solutions:

  • Try different terminal emulator (PuTTY, SecureCRT, screen)
  • Check if Break key is mapped correctly
  • Try Ctrl+Pause/Break
  • Hold Mode button during boot (hardware method)
  • Try connecting from different computer

config.text Not Found

Problem: flash:config.text doesn’t exist

Solutions:

switch: dir flash:
# Look for alternate names:
# - config.cfg
# - startup-config
# - vlan.dat

Switch Won’t Boot After Rename

Problem: Stuck at ROMMON

Solution:

switch: dir flash:
switch: boot flash:c2960-lanbasek9-mz.150-2.SE11.bin
# Replace with actual IOS filename from dir output

Lost Configuration After Recovery

Problem: Forgot to restore config.old

Solution:

  • Restore from backup (if you have one)
  • Reconfigure from scratch (see Basic Setup)

Legal and Ethical Considerations

  • Only perform password recovery on devices you own or have authorization to access
  • Unauthorized access to network devices is illegal
  • Document all password recovery procedures
  • Maintain audit logs of administrative access
  • This guide is for legitimate recovery and authorized security testing only
