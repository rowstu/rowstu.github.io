Setting Up a New Cisco Switch

Step-by-step guide for configuring a brand new Cisco switch for basic switching operations.

Initial Connection

Connect to the switch via console cable:

# Using screen (Linux/Mac)
screen /dev/ttyUSB0 9600

# Using minicom (Linux)
minicom -D /dev/ttyUSB0

# Using PuTTY (Windows)
# Serial line: COM3 (check Device Manager)
# Speed: 9600

Console Settings:

  • Baud rate: 9600
  • Data bits: 8
  • Parity: None
  • Stop bits: 1
  • Flow control: None

Initial Configuration Dialog

On first boot, the switch will prompt:

Would you like to enter the initial configuration dialog? [yes/no]:

Type no to configure manually (recommended for learning).

Basic Configuration Commands

Enter Privileged Mode

Switch> enable
Switch#

Enter Global Configuration Mode

Switch# configure terminal
Switch(config)#

Set Hostname

Switch(config)# hostname CoreSwitch
CoreSwitch(config)#

Configure Enable Password

# Secret password (encrypted with MD5)
CoreSwitch(config)# enable secret YourSecurePassword123!

# Or enable password (older, less secure)
CoreSwitch(config)# enable password YourPassword

Note: Use enable secret - it’s encrypted by default. The enable password is stored in plain text unless service password-encryption is enabled.

Configure Management Interface

Assign IP to VLAN 1 (Default Management VLAN)

CoreSwitch(config)# interface vlan 1
CoreSwitch(config-if)# ip address 192.168.1.10 255.255.255.0
CoreSwitch(config-if)# no shutdown
CoreSwitch(config-if)# exit

Set Default Gateway

CoreSwitch(config)# ip default-gateway 192.168.1.1

Configure Console Access

CoreSwitch(config)# line console 0
CoreSwitch(config-line)# password ConsolePass123
CoreSwitch(config-line)# login
CoreSwitch(config-line)# logging synchronous
CoreSwitch(config-line)# exec-timeout 5 0
CoreSwitch(config-line)# exit

Parameters:

  • logging synchronous - Prevents log messages from interrupting commands
  • exec-timeout 5 0 - Session timeout (5 minutes, 0 seconds)

Configure VTY Lines (Telnet/SSH)

CoreSwitch(config)# line vty 0 15
CoreSwitch(config-line)# password VtyPass123
CoreSwitch(config-line)# login
CoreSwitch(config-line)# transport input ssh
CoreSwitch(config-line)# exec-timeout 5 0
CoreSwitch(config-line)# exit

Note: transport input ssh restricts to SSH only (recommended). For telnet, use transport input telnet or transport input all.

Configure Interface Descriptions

CoreSwitch(config)# interface gigabitethernet 0/1
CoreSwitch(config-if)# description Connection to Router
CoreSwitch(config-if)# exit

CoreSwitch(config)# interface gigabitethernet 0/2
CoreSwitch(config-if)# description Server Room Connection
CoreSwitch(config-if)# exit

Configure Port Security Basics

CoreSwitch(config)# interface gigabitethernet 0/5
CoreSwitch(config-if)# switchport mode access
CoreSwitch(config-if)# switchport port-security
CoreSwitch(config-if)# switchport port-security maximum 2
CoreSwitch(config-if)# switchport port-security violation restrict
CoreSwitch(config-if)# switchport port-security mac-address sticky
CoreSwitch(config-if)# exit

Save Configuration

# Save running config to startup config
CoreSwitch# copy running-config startup-config
# Or shorthand
CoreSwitch# write memory
# Or even shorter
CoreSwitch# wr

Verify Configuration

# Show running configuration
CoreSwitch# show running-config

# Show startup configuration
CoreSwitch# show startup-config

# Show interface status
CoreSwitch# show ip interface brief

# Show VLAN information
CoreSwitch# show vlan brief

# Show MAC address table
CoreSwitch# show mac address-table

# Show version
CoreSwitch# show version

Essential Configuration Checklist

  • Hostname configured
  • Enable secret password set
  • Console password set
  • VTY password set
  • Management IP assigned
  • Default gateway configured
  • Interface descriptions added
  • Configuration saved to NVRAM
  • Remote access tested (if applicable)
  • Documentation updated

Common Post-Setup Tasks

Disable Unused Interfaces

CoreSwitch(config)# interface range gigabitethernet 0/10-24
CoreSwitch(config-if-range)# shutdown
CoreSwitch(config-if-range)# description UNUSED
CoreSwitch(config-if-range)# exit

Set Domain Name (Required for SSH)

CoreSwitch(config)# ip domain-name example.com

Configure Time and Date

# Set clock manually
CoreSwitch# clock set 14:30:00 6 Feb 2026

# Or configure NTP
CoreSwitch(config)# ntp server 192.168.1.1

Configure Banner

CoreSwitch(config)# banner motd #
***********************************************
* WARNING: Unauthorized access prohibited!   *
* All activities are monitored and logged.   *
***********************************************
#

Troubleshooting Common Issues

Cannot Connect via Console

  • Check baud rate (9600)
  • Try different USB port
  • Check cable (Cisco console cable vs null modem)
  • Press Enter several times to activate

No Network Connectivity

# Check interface status
show ip interface brief

# Ensure VLAN 1 is up
show vlan brief

# Verify at least one port is in VLAN 1 and up
show interfaces status

Forgotten to Save Config

# Compare running vs startup
show archive config differences system:running-config system:startup-config

# If changes exist, save them
copy running-config startup-config

Best Practices

  1. Always use enable secret instead of enable password
  2. Save configuration immediately after making changes
  3. Document everything - use interface descriptions
  4. Use strong passwords - minimum 8 characters with complexity
  5. Disable unused ports - security best practice
  6. Set appropriate timeouts - balance security and usability
  7. Use SSH instead of Telnet - encrypted communications
  8. Configure NTP - accurate timestamps for logs
  9. Set timezone - for accurate logging
  10. Regular backups - save configurations off-switch
