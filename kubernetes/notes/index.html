Architecture Overview

Kubernetes follows a master-worker architecture:

Control Plane (Master)

  • kube-apiserver: REST API endpoint
  • etcd: Distributed key-value store
  • kube-scheduler: Assigns pods to nodes
  • kube-controller-manager: Manages controllers (replicas, nodes, endpoints)

Worker Node

  • kubelet: Agent that runs containers
  • kube-proxy: Networking proxy for services
  • Container runtime: Docker, containerd, etc.

A. Deployment Strategies

Rolling Update (Default)

spec:
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxUnavailable: 1
      maxSurge: 1

Blue/Green Deployment

  • Deploy new version to new set of pods
  • Switch traffic using service selectors or ingress

Canary Deployment

  • Deploy 10% traffic to new version
  • Monitor metrics
  • Increase gradually

✅ 4. Networking

Kubernetes Networking Model

  • Every pod gets its own IP
  • No NAT between pods
  • Services provide stable endpoints
  • Network plugins implement networking (CNI)

Ingress & Egress

Ingress

apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: web-ingress
spec:
  rules:
  - host: myapp.example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: web-service
            port:
              number: 80

Egress

Control outbound traffic using:

  • Egress gateways
  • Network policies
  • Firewall rules

Network Policies

apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-nginx
spec:
  podSelector:
    matchLabels:
      app: nginx
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: frontend

Security

RBAC (Role-Based Access Control)

Example: Create Role

apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  namespace: default
  name: pod-reader
rules:
- apiGroups: [""]
  resources: ["pods"]
  verbs: ["get", "watch", "list"]

Bind Role to User

apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: read-pods
subjects:
- kind: User
  name: alice
  apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: Role
  name: pod-reader
  apiGroup: rbac.authorization.k8s.io

Network Security Practices

  • Use Network Policies

  • Use Private clusters

Scaling & Performance

Scaling Best Practices

  • Use Horizontal Pod Autoscaler (HPA)
  • Use Cluster Autoscaler
  • Set resource requests/limits

Example: HPA

kubectl autoscale deployment myapp --cpu-percent=50 --min=2 --max=10

Performance Tuning Tips

  • Avoid overcommitment of CPU and memory

  • Use node taints and pod tolerations for specialized workloads

  • Use QoS classes:

    • Guaranteed
    • Burstable
    • BestEffort

  • Use readiness probes to prevent traffic to unready pods

  • Use liveness probes to restart unhealthy pods

Troubleshooting

Common Issues & Solutions

1. Pod CrashLoopBackOff

kubectl describe pod <pod>
kubectl logs <pod> --previous

Fix: Check startup scripts, missing environment variables, failing dependencies.

2. Service Not Accessible

kubectl get svc
kubectl describe svc <service>
kubectl get endpoints

Fix: Ensure pod labels match service selector.

3. DNS Not Working

kubectl exec -it <pod> -- nslookup kubernetes.default

Fix: Check CoreDNS pods, configmaps, and network plugin.

4. Node Not Ready

kubectl describe node <node>

Fix: Check kubelet status, disk pressure, network issues.
